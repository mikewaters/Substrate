# My Browser Bookmarks Structure

Related: Consolidate my Bookmarks (Objective)

### Objectives

Create a defined structure for URL storage, that is used across web browsers, Raindrop, and other tools. Utilize defined LifeOS Ontology where possible.

### Principles

Not every object that has an associated URL needs to be stored in this way; they can be retrieved and stored elsewhere, with the URL as a reference; only if I need to access that URL in the future should I actually be capturing it in a bookmark.

## Browser bookmarks

Meant more for locality-of-reference, where the things I need to *use* are close at hand and easy to find. When I need to use a web browser for some activity, I need that activity‚Äôs required resources to be findable.

- Use/purpose: Regular activities, like writing code, doing work, researching, paying a bill, etc

- Attributes: Low frequency of storage, higher frequency of use

- Organizing principles: Easy to find without using metadata, and easy to store; requires simplicity and predictability

- System attributes:

   - Structured hierarchical classification (aka folders) to make finding straightforward

## Bookmarks Manager

[Raindrop.io.md](./Raindrop.io.md)

More of content and reference storage, attached to a personal interest that might be needed for a project some day, or to some short or long term effort I am engaging in.

- Use/purpose: Warm and cold storage of useful content, which I could not trivially find myself if I needed to search the internet

- Attributes: High frequency of storing, low frequency of use

- Organizing principles: Easy to save while still attaching the required metadata to surface later for some effort, or allow an agent to proactively associate to that effort

- System attributes:

   - Long-term caching for URL resources

   - Unstructured classification (aka tags, folders etc), to make search easier.

   - Ability to make notes for classification or retrieval later
